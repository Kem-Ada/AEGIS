# Contributing to AEGIS

Thank you for your interest in contributing to AEGIS — AI Ethics & Governance Integrated System. 🎉

AEGIS is a community-driven open source project. Whether you're an AI security professional, governance practitioner, machine learning engineer, or developer — your contributions are welcome.

---

## How to Contribute

### 1. Report Bugs
Found something that isn't working? Open an [Issue](https://github.com/Kem-Ada/AEGIS/issues) and include:
- What you were trying to do
- What happened instead
- Steps to reproduce the issue

### 2. Suggest Features
Have an idea for a new module or improvement? Open an Issue with the label `enhancement` and describe:
- The problem you're trying to solve
- Your proposed solution
- Any relevant regulatory or technical context

### 3. Submit Code
1. Fork the repository
2. Create a new branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Commit with a clear message: `git commit -m "Add: EU AI Act Article 13 transparency module"`
5. Push to your fork: `git push origin feature/your-feature-name`
6. Open a Pull Request

## Areas We Need Help With

We especially welcome contributions in these areas:

- 🔐 **AI security** — adversarial threat modelling, model robustness assessment, supply chain risk
- 👁️ **Human-in-the-loop** — oversight patterns, escalation frameworks, audit trail design
- 💡 **Explainability** — interpretability tooling, transparency report generation
- 🗺️ **Regulatory mappings** — NIST AI RMF, ISO 42001, UK AI Code of Practice, DORA
- ☁️ **Cloud integrations** — AWS, Azure, GCP deployment documentation
- 🧪 **Testing** — unit tests, integration tests, test coverage improvements
- 📖 **Documentation** — guides, tutorials, worked examples for real-world use cases

---

## Good First Issues

New to open source? Look for issues labelled `good-first-issue` — these are smaller, well-scoped tasks that are a great way to get started.

---

## Code Standards

- Python 3.9+
- Follow PEP 8 style guidelines
- Add docstrings to all functions and classes
- Include comments for complex logic
- Keep pull requests focused — one feature or fix per PR

---

## Community Standards

AEGIS is an inclusive, professional community. We expect all contributors to:

- Be respectful and constructive in all interactions
- Welcome contributors of all experience levels
- Give credit where it's due
- Focus on what's best for the project and its users
